# Learning Steps Index

このディレクトリは、`LEARNING_ROADMAP.md` をさらに具体化したステップ別ガイドです。

読む順番は上から順番です。途中を飛ばすより、各ステップで「動く状態」を作ってから次に進む方が早いです。

## 読む順番

1. `00_overview.md`: 全体像、開発の進め方、確認方法。
2. `01_hello_world.md`: テンプレートを Research Note 用に変える。
3. `02_command_group.md`: `/research` コマンド群を作る。
4. `03_json_storage.md`: JSON 保存を作る。
5. `04_keyword_search.md`: キーワード検索を作る。
6. `05_llm_answer.md`: LLM に資料を渡して回答させる。
7. `06_config_and_safety.md`: 設定、安全処理、削除確認を入れる。
8. `07_refactor.md`: コードをファイル分割する。
9. `08_embedding_next.md`: embedding 検索へ進む。

## 先に読むと楽になる補助資料

コードの前提知識が分からないときは、以下を先に読んでください。

1. `../concepts/README.md`: 補助資料の索引。
2. `../concepts/01_python_for_plugins.md`: プラグイン開発で必要な Python 基礎。
3. `../concepts/02_astrbot_plugin_terms.md`: AstrBot の用語と仕組み。
4. `../concepts/03_yield_async_event.md`: なぜ `yield` や `async/await` が出てくるのか。
5. `../concepts/04_rag_terms.md`: RAG、embedding、ベクトル検索などの用語。
6. `../concepts/05_developer_workflow.md`: 開発者の調べ方、読み方、デバッグ方法。
7. `../concepts/06_code_english_glossary.md`: コード中の英語、変数名、関数名の意味。

## 基本ルール

- 1ステップずつ進める。
- 1ステップ終わるたびに AstrBot を起動して試す。
- 変更後は `git status` と `git diff` を見る。
- エラーが出たら、まず一番下のエラーメッセージを見る。
- 完璧に理解するより、まず小さく動かす。

## コード理解の読み方

各ステップには、実装手順だけでなく以下の説明も追加しています。

- コードを詳しく読む: サンプルコードの意味を1つずつ説明します。
- 周辺知識: そのコードを理解するために必要な Python、AstrBot、アプリ開発の考え方を説明します。
- 開発者の考え方: なぜその順番で作るのか、なぜその設計にするのかを説明します。

読むときは、コードを暗記しようとしなくてよいです。以下の3つを意識してください。

```text
このコードはいつ呼ばれるのか
このコードは何を入力として受け取るのか
このコードは何を出力または保存するのか
```

この3つが分かれば、コードを自分で変更できるようになります。

## 本物の開発者になるための練習

各ステップで、ただ写すだけではなく以下を試してください。

- 変数名を見て、何を表しているか説明する。
- 関数の最初と最後を見て、入力と出力を説明する。
- エラーが出たら、すぐ直そうとせずエラー文を読む。
- 動いたら `git diff` を見て、自分が何を変えたか説明する。
- 1つ変更したら、すぐ起動して確認する。

開発者になるとは、すべてを暗記することではありません。分からないものを分解し、調べ、少しずつ動く形にできることです。

## 完成目標

最初の完成目標は `v0.1.0` です。

- `/research help` が使える。
- `/research add <text>` で資料を保存できる。
- `/research list` で資料一覧を見られる。
- `/research ask <question>` で資料に基づく回答が返る。
- `/research clear --confirm` で資料を削除できる。
- 保存は JSON。
- 検索はキーワード検索。
