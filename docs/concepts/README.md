# Concepts Index

このディレクトリは、Research Note を作るために必要な周辺知識をまとめた場所です。

ステップ別ガイドを読んでいて、以下のように感じたらここを読んでください。

- `yield` が急に出てきて意味が分からない。
- `@filter.command` の `@` が何か分からない。
- `async` と `await` がなぜ必要なのか分からない。
- `event`、`Context`、`Star`、`Provider` など AstrBot 用語が分からない。
- RAG、embedding、ベクトルDBなどの専門用語が分からない。
- どの順番でコードを読めばいいか分からない。

## 読む順番

1. `01_python_for_plugins.md`: プラグイン開発で必要な Python 基礎。
2. `02_astrbot_plugin_terms.md`: AstrBot の用語と仕組み。
3. `03_yield_async_event.md`: `yield`、`async/await`、イベント処理。
4. `04_rag_terms.md`: RAG と NotebookLM 風機能の用語。
5. `05_developer_workflow.md`: 開発者の作業方法、調べ方、デバッグ方法。
6. `06_code_english_glossary.md`: コード中によく出る英語の意味。

## 大事な考え方

最初から全部理解する必要はありません。

おすすめの読み方です。

1. ステップ別ガイドを読む。
2. 分からない言葉が出たら concepts を読む。
3. もう一度ステップ別ガイドに戻る。
4. 小さくコードを動かす。

プログラミングは、知らない言葉を少しずつ減らしていく作業です。
